A BlockingQueue in Java is a thread-safe queue that blocks when you try to add an element to a full queue or retrieve an element from an empty queue. It is part of the java.util.concurrent package and is commonly used for producer-consumer scenarios.

Here’s a detailed breakdown:

Key Points

  • Interface: java.util.concurrent.BlockingQueue<E>

  • Thread-safe: No need to manually synchronize access.

  • Blocking behavior:

    • put(E e): waits if the queue is full.

    • take(): waits if the queue is empty.

  • Useful for: producer-consumer, task scheduling, inter-thread communication.

Common Implementations

ImplementationCharacteristics
ArrayBlockingQueueBounded, backed by an array (fixed size).
LinkedBlockingQueueCan be bounded or unbounded, backed by linked nodes.
PriorityBlockingQueueOrders elements according to natural ordering or comparator.
DelayQueueElements can only be taken after a delay.
SynchronousQueueNo capacity, each insert waits for a remove.
LinkedTransferQueueOptimized for producer-consumer with transfer semantics.

Key Methods

MethodDescription
put(E e)Inserts element, waits if necessary for space.
take()Retrieves and removes head, waits if empty.
offer(E e, long timeout, TimeUnit unit)Tries to insert within given time.
poll(long timeout, TimeUnit unit)Tries to retrieve within given time.
remainingCapacity()Returns number of additional elements that can be inserted (if bounded).

Example: Producer-Consumer Using BlockingQueue

java
import java.util.concurrent.*;

public class BlockingQueueExample {
    public static void main(String[] args) {
        BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(5);

        // Producer
        Runnable producer = () -> {
            try {
                for (int i = 1; i <= 10; i++) {
                    queue.put(i); // waits if full
                    System.out.println("Produced: " + i);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        };

        // Consumer
        Runnable consumer = () -> {
            try {
                for (int i = 1; i <= 10; i++) {
                    int value = queue.take(); // waits if empty
                    System.out.println("Consumed: " + value);
                }
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        };

        new Thread(producer).start();
        new Thread(consumer).start();
    }
}

What happens here:

  • The producer will wait if the queue is full.

  • The consumer will wait if the queue is empty.

  • No explicit synchronization is needed.
